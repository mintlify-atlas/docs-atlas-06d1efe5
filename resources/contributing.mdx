MiroFish is an open-source project, and we welcome contributions from the community. Whether you're fixing bugs, adding features, improving documentation, or sharing ideas, your help is appreciated.

## Getting Started

<Steps>
  <Step title="Fork the Repository">
    Fork [666ghj/MiroFish](https://github.com/666ghj/MiroFish) on GitHub
  </Step>
  
  <Step title="Clone Your Fork">
    ```bash
    git clone https://github.com/YOUR_USERNAME/MiroFish.git
    cd MiroFish
    ```
  </Step>
  
  <Step title="Set Up Development Environment">
    Follow the [installation guide](/installation) to set up your local environment
  </Step>
  
  <Step title="Create a Branch">
    ```bash
    git checkout -b feature/your-feature-name
    ```
  </Step>
  
  <Step title="Make Changes">
    Write your code, add tests, and update documentation as needed
  </Step>
  
  <Step title="Test Your Changes">
    Ensure all tests pass and the application runs correctly
  </Step>
  
  <Step title="Submit a Pull Request">
    Push your changes and open a PR with a clear description
  </Step>
</Steps>

## Development Guidelines

### Code Style

**Python (Backend)**
- Follow PEP 8 style guidelines
- Use type hints where appropriate
- Add docstrings to functions and classes
- Keep functions focused and modular

**JavaScript/Vue (Frontend)**
- Use ES6+ syntax
- Follow Vue.js style guide
- Use meaningful variable names
- Add comments for complex logic

### Commit Messages

Write clear, descriptive commit messages:

```
feat: add agent interview batch endpoint
fix: resolve memory leak in simulation runner
docs: update API reference for report generation
refactor: simplify graph builder service
```

### Pull Request Guidelines

<Accordion title="PR Checklist">
  - [ ] Code follows project style guidelines
  - [ ] Tests are included and passing
  - [ ] Documentation is updated
  - [ ] Commit messages are clear
  - [ ] PR description explains the changes
  - [ ] No merge conflicts with main branch
</Accordion>

**PR Description Template:**
```markdown
## Description
Brief description of changes

## Motivation
Why these changes are needed

## Changes Made
- List of specific changes

## Testing
How the changes were tested

## Screenshots (if applicable)
```
