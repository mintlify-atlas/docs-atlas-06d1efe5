## Custom Ontology Design

### Understanding Ontology

The **ontology** defines what entities and relationships MiroFish can extract from your seed data.

**Default Ontology** (simplified):
```json
{
  "node_types": [
    {"name": "Person"},
    {"name": "Organization"},
    {"name": "Event"},
    {"name": "Topic"}
  ],
  "edge_types": [
    {"name": "PARTICIPATES_IN"},
    {"name": "WORKS_FOR"},
    {"name": "DISCUSSES"},
    {"name": "SUPPORTS"},
    {"name": "OPPOSES"}
  ]
}
```

### Applying Custom Ontology

**Via API:**

```bash
curl -X POST http://localhost:5001/api/graph/ontology/generate \
  -F "files=@my_document.pdf" \
  -F "simulation_requirement=Analyze financial market reactions" \
  -F "additional_context={\"ontology\": {\"node_types\": [...], \"edge_types\": [...]}}"
```

**Via Code:**

```python
from app.services.ontology_generator import OntologyGenerator

ontology_gen = OntologyGenerator()

custom_ontology = {
    "node_types": [
        {"name": "Investor"},
        {"name": "Company"}
    ],
    "edge_types": [
        {"name": "INVESTS_IN"}
    ]
}

result = ontology_gen.generate_ontology(
    text=document_text,
    simulation_requirement="Predict market reactions",
    custom_ontology=custom_ontology
)
```

## Custom Agent Behaviors

### Modifying Persona Generation

**Service**: `backend/app/services/oasis_profile_generator.py`

**Customization Points:**

1. **Persona Prompt Template**

Edit the LLM prompt that generates agent personas:

```python
# oasis_profile_generator.py (line ~150)

PERSONA_PROMPT_TEMPLATE = """
You are generating a persona for a social media simulation.

Entity Information:
- Name: {entity_name}
- Type: {entity_type}
- Relationships: {relationships}
- Background: {background}

Generate a detailed 2000-word persona including:
1. **Demographics**: Age, occupation, location
2. **Personality**: OCEAN traits, communication style
3. **Interests**: Topics they care about
4. **Opinions**: Stances on key issues
5. **Behavioral Patterns**: Activity level, posting style

--- YOUR CUSTOM INSTRUCTIONS HERE ---

Output as JSON.
"""
```

2. **Activity Probabilities**

Adjust how active agents are:

```python
# Default configuration
activity_config = {
    "post_probability": 0.3,
    "comment_probability": 0.5,
    "vote_probability": 0.7,
    "follow_probability": 0.2
}

# High-activity configuration
activity_config = {
    "post_probability": 0.6,
    "comment_probability": 0.8,
    "vote_probability": 0.9,
    "follow_probability": 0.4
}
```

3. **Personality Traits**

Customize OCEAN (Big Five) personality distributions:

```python
import random

def generate_personality():
    return {
        "openness": random.uniform(0.4, 0.9),  # Favor more open agents
        "conscientiousness": random.uniform(0.3, 0.8),
        "extraversion": random.uniform(0.5, 1.0),  # More extraverted
        "agreeableness": random.uniform(0.2, 0.7),
        "neuroticism": random.uniform(0.1, 0.5)
    }
```

### Example: Financial Analyst Agents

FINANCIAL_PERSONA_TEMPLATE = """

## Simulation Configuration

### Custom Platform Settings

**Service**: `backend/app/services/simulation_config_generator.py`

**Example: High-Activity Simulation**

```json
{
  "env": {
    "env_type": "reddit",
    "max_rounds": 96,
    "agent_num": 100,
    "time_compression": 0.5
  },
  "agents": {
    "model": "gpt-4",
    "temperature": 0.8,
    "activity_multiplier": 1.5
  },
  "simulation": {
    "initial_events": [
      {
        "type": "post",
        "content": "Breaking: Company announces Q4 earnings beat",
        "subreddit": "r/investing"
      }
    ],
    "enable_network_effects": true,
    "viral_threshold": 10
  }
}
```

**Parameters Explained:**

| Parameter | Default | Purpose |
|-----------|---------|----------|
| `max_rounds` | 48 | Simulation duration (2 rounds = 1 hour) |
| `agent_num` | 50 | Number of agents |
| `temperature` | 0.7 | LLM creativity (higher = more diverse) |
| `activity_multiplier` | 1.0 | Scale all activity probabilities |
| `initial_events` | [] | Seed posts to start simulation |
| `viral_threshold` | 20 | Upvotes needed for content to "go viral" |

### Custom Time Compression

Control simulation speed:

```python
# Real-time (1 round = 30 minutes)
time_compression = 1.0

# Faster (1 round = 15 minutes)
time_compression = 0.5

# Slower (1 round = 1 hour)
time_compression = 2.0
```

sentiment_tool = Tool(

# Add to ReportAgent

### 2. Custom Action Types (OASIS)

**Advanced**: Extend OASIS to support new actions.

<Warning>
This requires modifying the OASIS framework (not recommended for beginners).
</Warning>

**Example: Add "SHARE_ARTICLE" action**

```python
# In OASIS action space
class ShareArticleAction(Action):
    action_type = "SHARE_ARTICLE"
    
    def __init__(self, agent_id, article_url, comment):
        self.agent_id = agent_id
        self.article_url = article_url
        self.comment = comment
    
    def execute(self, environment):
        # Post article link with comment
        environment.create_post(
            author=self.agent_id,
            content=f"{self.comment}\n\n{self.article_url}"
        )
```

## Best Practices

<AccordionGroup>
  <Accordion title="Start Small">
    - Test customizations with small simulations first (20 agents, 12 rounds)
    - Validate ontology extraction before full simulation
    - Review generated personas for quality
  </Accordion>
  
  <Accordion title="Version Control">
    - Track custom prompts and configurations in Git
    - Document why you made specific customizations
    - Keep default configurations for comparison
  </Accordion>
  
  <Accordion title="Iterate Based on Results">
    - Review simulation outputs to refine personas
    - Adjust activity probabilities if agents too active/passive
    - Refine ontology if key entities are missed
  </Accordion>
  
  <Accordion title="Monitor Costs">
    - Custom high-activity configs increase LLM API costs
    - Use cheaper models (GPT-3.5) for testing
    - Estimate costs: agents × rounds × cost-per-call
  </Accordion>
</AccordionGroup>
